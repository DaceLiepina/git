Git and GitHub

Git is a distributed version control system (DVCS) used to track changes in source code during software development.

🌟 Why Use Git?

👥

Team Collaboration

  • Multiple developers can work simultaneously
  • Real-time synchronization of changes
🕰️

Version Control

  • Track all changes to codebase
  • Revert to previous versions easily
💾

Backup & Recovery

  • Full project history is stored
  • Protection against data loss
🌍

Open Source Friendly

  • Powers platforms like GitHub
  • Facilitates community contributions

✨ Key Features of Git

🖥️

Distributed System

  • Every developer has a full repository copy
  • Enables offline work and faster operations
🌿

Branching & Merging

  • Create branches for features/fixes
  • Seamless merging into main codebase
📜

Commit History

  • Every change recorded as a commit
  • Easy tracking and reverting
⏱️

Staging Area

  • Selective changes for commits
  • Better control over what gets committed
🤝

Collaboration-Friendly

  • Powers GitHub, GitLab, Bitbucket
  • Enables remote teamwork

🔧 Essential Git Commands

git init

Initialize a new Git repository

git add .

Stage all changes for commit

git commit -m "message"

Commit changes with a description

git status

Check untracked/modified files

git push

Upload changes to remote

git pull

Fetch changes from remote and merge into current branch

git log

View commit history with details (author, date, message)

git status

Show changed/untracked files and current branch stat

git remote -v

List all remote repositories (shows fetch/push URLs)

git reset

Undo changes (careful! can erase work)

GitHub is a cloud-based platform for hosting and collaborating on Git repositories. It provides tools for version control, code review, project management, and developer collaboration, making it one of the most widely used platforms for open-source and private software development.
